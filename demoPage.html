Some questions are better left unanswered. Luckily, you can create a simple true/false/null toggler for all those times when choosing NOT to decide is still a choice.
tfxStyles.css alters the thumb, and other colors, of a standard range type input to resemble a common toggle swtich - by setting the min to zero, the max to two, and the step to one, the slider will snap only to integer values.
tfxScript.js handles coloring the thumb to react to its being used - getting jammed in to a pumpkin, for instance, or perhaps simply sliding left and right. The script and css alter all inputs of the type range with the class "tfx tfx-slider"

Without any options, this HTML produces the following default slider: 

<input 
 type="range"
 class="tfx tfx-slider"
/>
Several options, set in the data-options param, allow customization of the true/false/null togglette. Pronounced "tahg-luh-ZHAY". Also, the script adjusts the slider and thumb to match style sizing. 

<input
 type="range"
 class="tfx tfx-slider"
 style="
     height: 50px;
     width: 100%;
 "
 data-options='{
    "value" : 2,
    "trueColor" : "#080",
    "falseColor" : "var(--samp-var)",
    "disabledColor" : "rgba(200,0,0,0.1)",
    "transSpeed" : "0.3s",
    "step" : 1,
    "disabled" : false
 }'
/>
Watch your head on that data-options parameter; it needs to be a well-formatted JSON string
